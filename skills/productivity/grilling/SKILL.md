---
name: grilling
description: Grill the user relentlessly about a plan or design. Use when the user wants to stress-test a plan before building, or uses any 'grill' trigger phrases.
---

Interview me relentlessly about every aspect of this plan until we reach a shared understanding. Walk down each branch of the design tree, resolving dependencies between decisions in the right order. For each question, provide your recommended answer.

Ask a small batch of questions when they are independent enough to answer now, and only split to one-at-a-time when a later question truly depends on the answer to an earlier one. Group related questions so the user can answer a sensible set in one pass.

If a _fact_ can be found by exploring the codebase, look it up rather than asking me. The _decisions_, though, are mine — put each one to me and wait for my answer.

Do not enact the plan until I confirm we have reached a shared understanding.
